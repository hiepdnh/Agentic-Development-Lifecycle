# Ask First Gates — 機微な変更リスト

以下の変更は **シニア / テックリードに確認するまで停止が必須** です。
このファイルが唯一の真実源 — `/dev:implement`、`/dev:pr`、`/dev:debug`、`/sec:review` から参照されます。

---

## リスト

| # | 変更種別 | 機微である理由 |
|---|---------------|----------------|
| 1 | 認証 / 認可ロジックの変更 | 失敗 = システム全体のセキュリティ喪失 |
| 2 | 権限 / ロールチェックの追加・修正 | ロール漏れ = 権限昇格 |
| 3 | パブリック API の破壊的変更 | 既存クライアント破壊、バージョニング計画必要 |
| 4 | データ損失可能性のあるデータベースマイグレーション | バックアッププラン無しでは復旧不能 |
| 5 | アクセス制御に影響する DB スキーマ変更 | 誤ったユーザーへのデータ露出可能性 |
| 6 | 共有インフラまたは設定の変更 | 稼働中の全サービスに影響 |
| 7 | 新規機微データ保存（PII、決済、健康情報） | コンプライアンス: APPI、PCI-DSS、GDPR |
| 8 | CORS 設定の変更 | 誤設定 = CSRF または本番フロントエンドブロック |
| 9 | 暗号実装の変更 | カスタム暗号 = ほぼ常に間違い |
| 10 | 新規外部 API 統合の追加 | 新規攻撃面の開放、セキュリティレビュー必要 |
