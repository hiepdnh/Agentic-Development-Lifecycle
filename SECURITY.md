# Security Policy

## Scope

This repository distributes a Claude Code skill pack — primarily Markdown command files, agent definitions, templates, and small Node.js installer scripts (`bin/install.js`, `bin/dashboard.js`). There is no runtime server, no user authentication, no handling of end-user data.

Security concerns within scope:
- Installer scripts writing files outside the intended target directory
- Templates or commands that could exfiltrate data or execute unintended code when rendered by Claude Code
- Supply-chain risks in declared npm dependencies

Out of scope:
- Issues in Claude Code itself — report at https://github.com/anthropics/claude-code/issues
- Project-level vulnerabilities in code generated by skills (those belong to the consuming project)

## Supported Versions

Only the latest released version of the framework receives security fixes. Older versions are not patched — users should re-run the installer to update.

| Version  | Supported          |
| -------- | ------------------ |
| latest   | :white_check_mark: |
| previous | :x:                |

## Reporting a Vulnerability

Please report security issues privately via GitHub Security Advisories on this repository:
https://github.com/hiepdnh/Agentic-Development-Lifecycle/security/advisories/new

Include:
- Affected file(s) and version / commit SHA
- Reproduction steps
- Impact assessment

Expect an initial response within 5 business days (JST). Confirmed issues are typically patched within 2 weeks; complex issues may take longer and we will keep you informed.

Do not open public GitHub issues for security reports.
